# 1. Бизнес-анализ (Business Understanding)

## Организационная структура
Делаю единолично.

## Бизнес-цель проекта
Сокращение времени дизайнеров на прототипирование визуального контента, соответствующего корпоративному стилю и бренд-буку, а также снижение стоимости креативного продакшена.

## Существующие решения и их ограничения
На рынке присутствует большое количество нейросетей, генерирующих изображения (например, **Midjourney**, **Kandinsky**).  
Однако:
- их визуальный стиль может не совпадать с потребностью компании, из-за чего требуется создание точных промтов, что затрудняет использование;
- отсутствует встроенная система оценки соответствия бренд-гайдам.

## 1.2. Решаемые задачи с точки зрения аналитики (Data Mining Goals)

### Метрики для оценки

- **CLIP Similarity Score** - степень соответствия изображения тексту (метрика с высокой чувствительностью к качеству промта идкально подходит под задачу).  
- **Aesthetic Score** - визуальная привлекательность.  
- **Человек** - мое экспертное мнение.

### Критерии успешности модели

- `CLIP score > 0.3`  
- `Aesthetic score > 5.5`  
- Минимум **70%** сгенерированных изображений проходят ручную проверку по бренд-гайду.

### **Бизнес-метрики (Business KPIs)**

| Метрика | Описание |
|-|-|
| **Сокращение времени на создание картинок** | Сравнение времени “ручного” прототипирования и работы через сервис |
| **Снижение нагрузки на дизайнеров** | Снижение среднего времени на создание контента за определеннный период |
| **Удовлетворенность пользователей** | Оценка удобства пользования сервисом |

### **Примерные подходы к реализации**

#### **Подход к улучшению промтов**

В проекте будет использоваться многоступенчатая система отмизации текстового описания:

1. **Предварительная обработка текста**  
   - автоматическое добавление элементов бренд-бука (цветовая палитра, стиль и тд);
   - нормализация текста;
   - разбор структуры промта.

2. **Генерация вариантов улучшенного запроса**  
   - генерация нескольких вариантов улучшенного промта на основе исходного описания пользователя;  
   - планируется использовать открытые модели:  
     - *FLAN-T5 (small/base)* - для локального выполнения;  
     - при необходимости - *Mistral*.

3. **Автоматическое улучшение при низком качестве**  
   - если качество изображения низкое, модель автоматически улучшит промт, учитывая замечания

4. **Выбор лучшего промта**  
   - из n-го количества сгенерированных промтов выбираются те, которые дают лучшие результаты по метрикам

#### **Генерация изображений (CV-компонент)**

- Используется модель **Kandinsky** или **Stable Diffusion**.  
- Генерруется несколько вариантов изображений для каждого улучшенного промта.  
- При необходимости **LoRA-fine-tuning**.

#### **Оценка качества изображений (Judge-компонент)**

Для выбора лучшего изображения находятся две независимые оценки:

- **CLIP** - измеряет соответствие изображения тексту.  
- **Aesthetic Predictor** - предсказывает визуальную привлекательность.  
- Оценки объединяются в итоговую оценку качества.

#### **Сервисная часть**

- API реализуется на **FastAPI**.  
- Архитектура: пользовательский промт → улучшение → генерация вариантов → оценка → возврат лучшего результата.

## 1.3. План проекта (Project Plan)

**Этапы:**
> (Business Understanding → Data Understanding → Data Preparation → Modeling → Evaluation → Deployment)

**План действий:**

6. Создание базового интерфейса
